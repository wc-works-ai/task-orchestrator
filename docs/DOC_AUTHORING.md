# Doc Authoring (Concise Edition)

**Why**: Readers have 5 seconds. Brevity respects their time.

## Rules

- **One idea per line** — each bullet, one concept
- **Active voice** — "Use X" not "X should be used"
- **No preamble** — start with the point, not context
- **Abbreviate** — env var, repo, config (not full terms)
- **Numbers not words** — "5 rules" not "five rules"
- **Skip punctuation** — commas optional, periods unnecessary
- **Verb-first bullets** — "Check logs" not "You should check"

## Anti-Patterns

- ❌ Long paragraphs — use bullets
- ❌ "For example…" preamble — just show it
- ❌ Verbose transitions — cut them

## Example

**Bad**: In the scenario where you encounter an error condition, it is recommended that you examine the log files to diagnose what went wrong.

**Good**: Error? Check logs.
